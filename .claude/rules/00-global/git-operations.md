# Git操作規約

## コミットメッセージ

### 禁止事項

AIによる生成を示す記述を含めない：

- `Generated with Claude Code`
- `Co-Authored-By: Claude`
- `🤖` などAI関連の絵文字
- `AI生成`、`AIアシスト` などの表現

### 推奨フォーマット

```
<type>: <簡潔な説明>

<詳細な説明（必要に応じて）>
```

**type例:**
- `feat`: 新機能
- `fix`: バグ修正
- `refactor`: リファクタリング
- `docs`: ドキュメント
- `test`: テスト
- `chore`: その他

## 破壊的コマンドの実行禁止

### 絶対に事前確認なしに実行してはいけないコマンド

以下のコマンドは**未保存の作業を完全に失わせる**ため、**必ずユーザーの明示的な許可を得てから実行すること**：

| コマンド | 危険性 | 代替案 |
|---------|--------|--------|
| `git reset --hard` | 未コミットの変更を全て削除 | `git stash` または `git restore` |
| `git clean -f` | 未追跡ファイルを削除 | `git clean -n` で確認後、手動削除 |
| `git clean -fd` | 未追跡ファイル・ディレクトリを削除 | `git clean -n` で確認後、手動削除 |
| `git checkout .` | 全ての未コミット変更を破棄 | `git restore .` を確認付きで |
| `git restore .` | 全ての未コミット変更を破棄 | ファイル単位で `git restore <file>` |
| `rm -rf` | ファイル・ディレクトリを強制削除 | `rm -i` で確認、または `trash` コマンド |
| `git push --force` | リモートの履歴を上書き | `git push --force-with-lease` + 事前確認 |

### 実行前の必須手順

1. **ユーザーに提案**: 「〇〇を実行しますが、未保存の変更が失われる可能性があります。よろしいですか？」
2. **代替案の提示**: より安全な方法がある場合は提示
3. **明示的な許可**: ユーザーが「実行して」と明言するまで実行しない

### 安全な代替パターン

```bash
# ❌ 危険: 未コミット変更を全て削除
git reset --hard

# ✅ 安全: 変更を一時退避
git stash
# 後で復元: git stash pop

# ❌ 危険: 未追跡ファイルを全て削除
git clean -f

# ✅ 安全: 確認してから削除
git clean -n  # 削除対象を確認
# ユーザー確認後に: git clean -f

# ❌ 危険: 特定ファイルの変更を破棄
git checkout -- file.ts

# ✅ 安全: restore で明示的に
git restore file.ts  # ユーザー確認後のみ
```

## Pull Request

### 禁止事項

- AI関連の記述を含めない
- 自動生成を示す文言を含めない

### 推奨フォーマット

```markdown
## 概要
<変更の目的と背景>

## 変更内容
- <変更点1>
- <変更点2>

## テスト
- [ ] 動作確認済み
- [ ] テスト追加/更新済み（該当する場合）
```
