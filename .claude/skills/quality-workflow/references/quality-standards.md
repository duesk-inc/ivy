# 品質メトリクス基準

## コードカバレッジ目標

| 対象 | 目標 |
|------|------|
| バックエンド | 80%以上 |
| フロントエンド | 70%以上 |

## 複雑度目標

| メトリクス | 目標 |
|-----------|------|
| 関数あたりの行数 | 50行以下 |
| サイクロマティック複雑度 | 10以下 |

## 重複度目標

| メトリクス | 目標 |
|-----------|------|
| 重複コード | 5%以下 |

## 静的解析ツール

### Go
- `go vet` - 基本的な静的解析
- `golangci-lint` - 包括的なリンター

### TypeScript
- ESLint - コード品質チェック
- TypeScript コンパイラ - 型チェック

## セキュリティ基準

| カテゴリ | 要件 |
|---------|------|
| 入力検証 | すべての外部入力をバリデーション |
| 認証 | JWTトークン検証必須 |
| 認可 | RBACによる権限管理 |
| SQLインジェクション | パラメータバインディング必須 |
| XSS | 出力エスケープ必須 |
