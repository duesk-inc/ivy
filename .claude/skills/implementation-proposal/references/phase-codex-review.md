# Codex設計レビューフェーズ

## 目的
Codex（OpenAI）に実装計画の設計妥当性をレビューさせ、LGTMを得る。

**スコープ: 設計・方針レベルのレビューのみ。コードレビューは行わない。**

## ⚠️ 最重要ルール: レビュー結果の待機

**Codexへレビューを送信したら、レビュー結果のレスポンスを必ず受け取り、内容を解析してからでなければ、絶対に次のフェーズ（IMPLEMENT）に進んではならない。**

- `mcp__codex__codex` の戻り値にレビュー内容が含まれている → その内容を判定する
- Codexが確認応答（行動計画の提示等）のみを返した場合 → `mcp__codex__codex-reply` で「はい、レビューを実行してください」と返信し、**実際のレビュー結果を受け取るまで待つ**
- レスポンスに「LGTM」「問題なし」「approve」等のレビュー判定が含まれていない場合は、まだレビューが完了していない → 追加の応答を待つ
- **レビュー結果を受け取らずにIMPLEMENTに進むことは禁止**

## 前提条件
- プロジェクトルートに `AGENT.md` が存在すること
- `AGENT.md` にはプロジェクト概要、アーキテクチャ制約、レビュー観点が記載されている
- Codexは `sandbox: "read-only"` で起動し、`.claude/rules/` 配下のファイルを参照可能

## 実行手順

### 1. Codexセッション開始

`mcp__codex__codex` を使用してセッションを開始する。

```
mcp__codex__codex(
  prompt: "<review-prompt-templates.mdの設計レビューテンプレートを使用>",
  cwd: "<プロジェクトルート>",
  sandbox: "read-only"
)
```

**重要**:
- `cwd` を指定してCodexがプロジェクトファイルを読めるようにする
- `sandbox: "read-only"` でファイル読み取りのみ許可
- レスポンスの `conversationId` を必ず保持する

### 1.5. Codexの応答確認（必須）

Codexは最初の応答で行動計画の確認を求めることがある。その場合:

```
mcp__codex__codex-reply(
  conversationId: "<取得したID>",
  prompt: "はい、レビューを実行してください"
)
```

**この応答にレビュー判定（LGTM / 指摘事項）が含まれるまで繰り返す。**
レビュー判定が得られるまでは、次のステップに進まないこと。

### 2. レビュー結果の判定

Codexの応答を解析し、以下のいずれかに分類する：

| 判定 | Codex応答の特徴 | アクション |
|------|----------------|----------|
| **LGTM** | 「LGTM」「問題ありません」「approve」「承認」等 | IMPLEMENT へ進む |
| **NEEDS_FIX** | `[必須]` or `[推奨]` の指摘がある | 計画修正して再レビュー |
| **SUGGESTION_ONLY** | `[提案]` のみ | ユーザーに対応要否を確認 |
| **UNCLEAR** | 判定が曖昧 | ユーザーに判断を仰ぐ |

### 3. 修正→再レビューループ

`[必須]` または `[推奨]` の指摘があった場合：

1. Codexの指摘事項をユーザーに報告
2. 指摘に基づき実装計画を修正
3. `mcp__codex__codex-reply` で修正内容を同一セッションに送信

```
mcp__codex__codex-reply(
  conversationId: "<保持したID>",
  prompt: "<修正内容を含む再レビュー依頼>"
)
```

4. 再度レビュー結果を判定

### 4. 修正→再レビューの継続

- **上限回数なし**: LGTMを取得するまで修正→再レビューを繰り返す
- 各修正ラウンドでは、Codexの指摘をユーザーに報告し、修正後に再レビューを依頼する

## ユーザーへの報告フォーマット

### レビュー依頼時
```markdown
## Codex設計レビュー依頼中
Codexに実装計画のレビューを依頼しています...
（Codexは AGENT.md + プロジェクト規約を参照してレビューします）
```

### レビュー結果受領時
```markdown
## Codex設計レビュー結果
**判定**: [LGTM / 修正要求]
**修正回数**: N回目

### 指摘事項（修正要求の場合）
#### [必須]
1. [指摘内容]

#### [推奨]
1. [指摘内容]

#### [提案]
1. [指摘内容]
```

### LGTM取得時
```markdown
## Codex設計レビュー LGTM
**レビュー回数**: N回（修正M回）
**次のステップ**: 実装フェーズへ進みます
```
