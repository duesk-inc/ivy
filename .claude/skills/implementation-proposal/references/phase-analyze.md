# Phase 1: 要件分析 (ANALYZE)

## 目的

実装要件を整理し、設計に必要な情報を収集する。

## 分析項目

### 1. 機能要件の整理

- ユースケースの洗い出し
- 入出力の定義
- ビジネスルール

### 2. 非機能要件の整理

| 項目 | 確認ポイント |
|------|------------|
| パフォーマンス | 応答時間、同時接続数 |
| 可用性 | SLA、ダウンタイム許容 |
| セキュリティ | 認証、認可、監査 |
| スケーラビリティ | 将来の拡張見込み |

### 3. 制約条件の整理

- 技術的制約（既存システム、言語、フレームワーク）
- ビジネス制約（期限、予算、リソース）
- 運用制約（デプロイ方式、監視要件）

### 4. Serenaでの調査

```bash
# 既存実装パターンの分析
read_memory("coding_conventions")
get_symbols_overview("類似機能のディレクトリ")

# アーキテクチャ分析
find_symbol("関連するインターフェース")
find_referencing_symbols("拡張ポイント")
```

## チェックリスト

- [ ] ユースケースを洗い出した
- [ ] 非機能要件を確認した
- [ ] 制約条件を整理した
- [ ] 既存システムとの整合性を確認した
- [ ] 不明点をリストアップした

## 出力形式

```
status: SUCCESS | NEED_CLARIFICATION
requirements:
  functional: [機能要件リスト]
  non_functional: [非機能要件リスト]
constraints: [制約条件リスト]
questions: [不明点リスト]
```
