# Phase 3: 評価 (EVALUATE)

## 目的

各実装アプローチを多角的に評価し、推奨案を決定する。

## 評価マトリクス

| 評価項目 | 重み | アプローチ1 | アプローチ2 | アプローチ3 |
|---------|------|------------|------------|------------|
| 実装工数 | 20% | - | - | - |
| パフォーマンス | 25% | - | - | - |
| 保守性 | 20% | - | - | - |
| 拡張性 | 15% | - | - | - |
| セキュリティ | 10% | - | - | - |
| テスト容易性 | 10% | - | - | - |
| **合計** | **100%** | - | - | - |

## 評価基準

### 実装工数（5段階）

| スコア | 基準 |
|-------|------|
| 5 | 1日以内 |
| 4 | 2-3日 |
| 3 | 1週間 |
| 2 | 2週間 |
| 1 | 1ヶ月以上 |

### パフォーマンス（5段階）

| スコア | 基準 |
|-------|------|
| 5 | 最適化された実装 |
| 4 | 十分な性能 |
| 3 | 許容範囲 |
| 2 | 改善が必要 |
| 1 | 要件未達 |

## リスク分析

### 各アプローチのリスク

| リスク種別 | アプローチ1 | アプローチ2 | アプローチ3 |
|-----------|------------|------------|------------|
| 技術的リスク | - | - | - |
| スケジュールリスク | - | - | - |
| 運用リスク | - | - | - |

### リスク対策

- 高リスク項目には軽減策を提示
- フォールバック案を用意

## 出力形式

```
evaluation:
  scores:
    approach1: 4.2
    approach2: 3.8
    approach3: 3.5
  recommendation: "approach1"
  rationale: [選定理由]
  risks:
    - risk: [リスク内容]
      mitigation: [対策]
```
